# 鬼畜配音（谷歌翻译N遍）功能设计文档

## 1. 背景与目标

把一段**自带烧录字幕**的影视片段（典型素材：央视版《三国演义》），
通过「谷歌翻译来回翻 N 遍」让台词层层失真、荒腔走板，再用 TTS 重新配音，
并把新台词字幕**叠在原字幕上方**，最终产出一段「一本正经地胡说八道」的鬼畜视频。

笑点来源：机器翻译反复直译累积的误差，而**不是**大模型的「聪明翻译」。
LLM 会自我纠错、保持语义通顺，反而破坏笑点；所以核心刻意使用「笨」的
机器翻译引擎（Google 网页版）多轮硬翻。

### 理想效果（用户原话）

> 项目运行后上传视频 → 在系统里手工截取部分片段 → 自动翻译 N 遍 →
> 把视频语音转为翻译后的语音 → 翻译后字幕附在原字幕**上方**（不覆盖原字幕）→
> 要求语音和新字幕播放一致。

## 2. 已敲定的设计决策

| # | 决策点 | 选择 |
|---|---|---|
| 1 | 原台词来源（原字幕已烧死在画面里，无法直接取文字） | Whisper 转录音频，**并加「手工校对」步骤**让用户改错字 |
| 2 | 片段里原背景音乐/音效 | **保留原音（压低约 -15dB）+ 叠加新配音** |
| 3 | 新语音与原视频时长对齐 | **新语音用 `atempo` 拉伸/压缩到正好等于原句时长**，三者走同一时间轴天然同步 |
| 4 | 翻译引擎 | `deep-translator` 的 `GoogleTranslator`（免费、无 key，已是项目依赖） |
| 5 | TTS 引擎 | `edge-tts`（免费、无 key、中文音色自然） |
| 6 | 字幕格式 | ASS（可精确定位，靠高 `MarginV` 压在原字幕上方） |

## 3. 整体流程

```
上传视频
  → ① 网页播放 + 手工设 入点/出点（HTML5 <video> + 滑块）
  → ② ffmpeg 按入/出点裁出片段
  → ③ 提取音频 → Whisper 转录 → 带时间轴的原台词 segments
  → ④ 【校对】前端展示逐句，用户改错字后回传
  → ⑤ 【劣化器】每句 中→英→日→韩→…→中 翻 N 遍，台词跑偏
  → ⑥ 【TTS】每句鬼畜台词 → 中文语音
  → ⑦ 【对齐】每句语音 atempo 拉伸到=原句时长 → 拼成新人声轨
  → ⑧ 【混音】原音轨压低 + 叠新人声轨
  → ⑨ 【字幕】新台词 ASS 字幕，压原字幕上方，时间轴=原句时间轴
  → ⑩ ffmpeg：裁好的画面 + 新音轨 + 烧新字幕 → 输出 mp4
  → ⑪ 任务完成，可下载 / 打开文件位置
```

**同步保证**：新字幕时间轴 = 原句时间轴；新语音被拉伸到同一时长。
画面、语音、字幕三者全程对齐，满足「语音和新字幕播放一致」。

## 5. 关键技术细节

### 5.1 劣化器（degrader.py）

- 轴心语言链（pivot），例如：`en, ja, ko, ar, ru, fr, de, hi`，循环使用。
- 第 i 遍：`GoogleTranslator(source=上一语言, target=下一语言).translate(text)`。
- N 遍后**强制翻回 `zh-CN`**，保证最终是中文。
- 失败/限流的容错：单步异常则跳过该步、保留上一步文本，继续。
- 每句之间 sleep 一点，避免 Google 网页接口限流。

### 5.2 TTS + 对齐（tts.py）

- 每句 `edge-tts` 合成出独立音频，得到该句自然时长 `d_i`。
- 目标时长 = 原句 `end_i - start_i`。计算语速倍率 `r = d_i / target`，
  用 ffmpeg `atempo=r`（>2 或 <0.5 需链式串联）拉伸到 target。
- 在一条静音底轨上，把每句按 `start_i` 位置贴进去 → 完整人声轨。
- 语音偶尔被加速反而强化鬼畜感，是预期效果。

### 5.3 混音（mixer.py）

- `ffmpeg -i clip.mp4 -i voice.wav -filter_complex
  "[0:a]volume=-15dB[a0];[a0][1:a]amix=inputs=2:duration=first[aout]"
  -map 0:v -map "[aout]" -c:v copy out.mp4`
- 原音压低，新配音叠加在上。

### 5.4 ASS 字幕定位（burner.py 扩展）

- 原字幕在底部居中（约画面底部）。新字幕用 ASS：
  - `Alignment=2`（底部居中）+ 较大 `MarginV`（如 80~120px）把它顶到原字幕上方；
  - 字号/描边可配置，便于区分原字幕。
- 时间轴直接用原句 `start/end`。

### 5.5 网页/裁剪/校对 UI（dub.html）

- 上传后用 `<video>` 加载（前端本地预览，URL.createObjectURL）。
- 入/出点：双滑块或「设为入点/出点」按钮，读 `video.currentTime`。
- 提交 → `/api/dub/process` 起任务，跑到转录后**暂停**，返回逐句文本。
- 校对面板可编辑逐句 → `/api/dub/confirm` 回传 → 继续后续阶段。
- 进度轮询 `/api/dub/status/<task_id>`，复用现有阶段展示。

## 6. 阶段任务进度的 stage 命名

`queued → trimming → extracting_audio → transcribing → awaiting_review →
degrading → synthesizing → aligning → mixing → burning → done`（或 `error`）。

## 8. 开发顺序（每步可单独验证）

1. **degrader.py** — 先验「翻 N 遍」的搞笑味道（核心、风险最高）。
2. **tts.py** — 合成语音 + 填时间槽 + 混音。
3. **ASS 字幕定位** — 新字幕稳稳压原字幕上方。
4. **dub.html + 路由 + _run_dub** — 串成完整 `/dub` 流程。

## 9. 风险与备注

- 三国台词为文言腔 + 古人名，Whisper `small` 易转错 → 已用「手工校对」缓解；
  必要时可升级到 `medium` 模型。
- Google 网页翻译接口可能限流；需限速 + 容错。
- 极端语速拉伸（句子很短但译文很长）会让语音很快或失真；可设倍率上限并截断。
- 版权：仅供个人学习娱乐，注意素材使用边界。
