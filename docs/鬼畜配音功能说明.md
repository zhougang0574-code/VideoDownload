# 鬼畜配音（谷歌翻译N遍 + 重新配音）功能说明 / 维护文档

> 面向后续维护者（含 Claude Code）。读完这份能直接定位每个环节的代码、知道
> 哪里改什么、以及目前哪些没做完。配套设计稿见 `docs/鬼畜配音设计.md`。

## 1. 一句话概括

上传一段**自带烧录字幕**的影视片段（典型：央视《三国演义》），在网页里截取一段，
台词经「Google 翻译来回翻 N 遍」变得荒腔走板，再用 TTS 重新配音，新台词字幕
叠在原字幕**上方**（不覆盖），语音/字幕/画面走同一时间轴保持同步。

## 3. 用户操作流程

1. `python main.py` → 浏览器开首页 → 点「鬼畜配音」进 `/dub`
2. 选视频文件 → `<video>` 本地预览
3. 播放到某处点「设为入点 / 设为出点」截取片段
4. 选「翻译遍数(5–40，默认20)」「配音引擎」「(edge时)预设音色」「是否保留原背景音」；
   云端克隆引擎需先在「云端 Key 配置」里填 Key 才会点亮
5. 点「开始处理」→ 上传并起任务
6. 转录完成后**自动暂停**，弹出逐句**校对面板**，改掉错字
7. 点「确认并开始鬼畜翻译」→ 放行后续流水线
8. 完成后给出「播放/下载」「打开文件位置」链接

## 4. 后端流水线（`webapp.py: _run_dub`）

阶段（即 task["stage"]，前端据此显示文案）：

```
queued → trimming → extracting_audio → transcribing
       → awaiting_review (阻塞，等校对放行)
       → degrading → synthesizing → mixing → burning → done
（任何异常 → error，task["error"] 存消息）
```

对应代码步骤：

1. `trim()` 裁片段 → `dub_tmp/<task_id>/clip.mp4`
2. `extract_audio()` → `audio.wav`
3. `transcribe()` → `segments`（相对片段的时间轴，从 0 开始）
4. 写入 `task["segments"]`，stage=`awaiting_review`，`DUB_EVENTS[task_id].wait()` 阻塞
5. `/api/dub/confirm` 写回校对后的 segments 并 `event.set()` 放行
6. `degrade_segments()` 翻 N 遍
7. `synth_segments()` 合成 + 对齐 → `voice.wav`
8. `mix_into_video()` 原音压低 + 叠配音 → `mixed.mp4`
9. `write_ass()` + `burn_ass()` 烧新字幕 → `dubbed_videos/鬼畜配音_<task_id>.mp4`
10. `finally` 删上传源文件 + 整个 `dub_tmp/<task_id>` 工作目录

**暂停机制**：任务在「转录完」处用 `threading.Event` 阻塞等待前端校对。
Event 存在 `DUB_EVENTS`（独立字典），**不能**放进 `TASKS`，因为 `TASKS[id]`
会被 `jsonify` 序列化，Event 不可序列化。

## 5. 模块与关键函数

### `subtitle/degrader.py` — 台词劣化器
- `degrade_text(text, rounds=20, langs=None, delay=0.3, on_step=None)`
  一句中文在 `PIVOT_LANGS` 间循环机翻 `rounds` 遍，最后强制翻回 `zh-CN`。
- `degrade_segments(segments, rounds, langs, delay, on_progress)`
  对每段只改 `text`、保留 `start/end`。
- 引擎：`deep_translator.GoogleTranslator`（免费、无 key）。`_translate` 带重试。
- 失败容错：单步异常跳过该步、保留上一步文本；错误日志**不打印外文原文**
  （否则 Windows GBK 控制台会再抛 `UnicodeEncodeError`）。
- 调味参数：`PIVOT_LANGS`（轴心语言链）、`rounds`。语言越「离谱」、遍数越多，
  跑偏越狠。

### `subtitle/voice_engines.py` — 可插拔配音引擎（音色克隆的入口）
- 统一接口 `VoiceEngine`：`available()` / `synth(text, out_path, ref_audio, voice)` /
  `synth_batch(jobs, on_progress)`；属性 `supports_clone`、`output_ext`、`needs`。
- 实现：`EdgeEngine`(预设,永远可用)、`XttsEngine`(本地克隆,装了 venv 才可用,
  重写 `synth_batch` 一次跑完)、`CosyVoiceEngine`/`IflytekEngine`/`VolcEngine`
  (云端克隆,填了 Key 才可用，网络调用标 `# 待验证`)。
- 注册表：`get_engine(id)`、`list_engines()`(给前端点亮)、`configured_providers()`。
- Key 存储：`voice_config.json`（gitignored）；`load_voice_config/save_voice_config`。
- **想加新引擎或做音色克隆：在这里加一个 `VoiceEngine` 子类即可，上层不用改。**

### `subtitle/tts.py` — 对齐 + 混音 + 驱动引擎
- `probe_duration(path)` 跑 ffmpeg 读 time，不依赖 ffprobe。
- `fit_to_duration(in_path, target_sec, out_path)` 用 `atempo` 把语音拉到目标
  时长；倍率超 [0.5,2.0] 自动链式串联，并受 `TEMPO_MIN/MAX` 限幅。
- `slice_reference(clip_audio, start, end, out_path, min_len=6.0)` 从片段原声切出
  某句对应参考音（两侧扩到≥min_len），供克隆引擎复刻该句说话人。
- `synth_segments(segments, out_path, engine="edge", voice, clip_audio, work_dir,
  on_progress)` 选引擎→（克隆引擎逐句切参考音）→`engine.synth_batch` 合成→逐句
  `fit_to_duration` 对齐→`adelay`+`amix` 拼成完整人声轨。**引擎无关的对齐/拼轨复用。**
- `mix_into_video(video_path, voice_wav, out_path, keep_bg=True, orig_db=-15)`
  keep_bg=True：原音 `volume=-15dB` + 新配音 `amix`；False：直接替换音轨。
- `synth_one()` 保留为 edge 单句封装（供 `__main__` 自测）。
- `DEFAULT_VOICE = "zh-CN-YunjianNeural"`（仅 edge 用）。

### `tts_clone/xtts_worker.py` — 本地 XTTS 子进程 worker
- 在独立 venv `tts_clone_venv` 里运行，`python xtts_worker.py jobs.json`，一次加载
  模型跑完所有 job，每个 job 用自己的 `ref_audio` 零样本克隆。安装步骤见文件头注释。

### `subtitle/clipper.py` — 裁剪
- `trim(video_path, start, end, out_path, on_progress)` `-ss/-t` 重编码裁切
  （重编码保证帧精确；直接 copy 会从最近关键帧开始多出一截）。

### `subtitle/burner.py` — 字幕（本功能新增 ASS 部分）
- `probe_resolution(video_path)` 用 `av` 读宽高；ASS 的 PlayResX/Y 要等于真实
  分辨率，`MarginV` 才能精确对应像素。
- `write_ass(segments, ass_path, video_width, video_height, margin_v=None,
  font_size=None, font_name, primary_colour, outline)`
  Alignment=2（底部居中）+ 较大 `MarginV`（默认 `height*0.12`）顶到原字幕上方；
  默认黄色字（`&H0000FFFF`）与原白字区分；字号默认 `height*0.05`。
- `burn_ass(video_path, ass_path, output_path, duration, on_progress)`
  用 ffmpeg `ass` 滤镜烧录，视频重编码、音频 `copy`。
- （原有 `write_srt/burn_subtitles` 保持不变，供「视频字幕」功能用。）

## 6. HTTP 接口（均在 `webapp.py`）

| 方法 路径 | 作用 | 入参 | 出参 |
|---|---|---|---|
| GET `/dub` | 页面 | — | HTML |
| POST `/api/dub/process` | 起任务（multipart） | `video` 文件, `start`, `end`, `rounds`, `voice`, `keep_bg` | `{task_id}` 或 `{error}` |
| POST `/api/dub/confirm` | 提交校对、放行 | JSON `{task_id, segments:[{start,end,text}]}` | `{ok}` 或 `{error}` |
| GET `/api/dub/status/<task_id>` | 轮询进度 | — | 整个 task 字典（含 stage/percent/segments/error/output_path） |
| GET `/dubbed_videos/<filename>` | 取成品 | — | 视频文件 |
| GET `/api/dub/engines` | 列引擎+可用性 | — | `{engines:[{id,name,supports_clone,available,needs}], configured:{...}}` |
| POST `/api/dub/voice-config` | 存云端 Key | JSON `{provider, fields:{...}}` | `{ok}` 或 `{error}` |
| POST `/api/open-location` | 资源管理器定位（已加 `DUB_OUTPUT_DIR` 白名单） | `{path}` | `{ok}` |

`/api/dub/process` 新增表单字段 `engine`（引擎 id，默认 `edge`）；后端会校验该
引擎 `available()`，不可用直接 400。

校对接口只采纳 `text` 改动，`start/end` 用前端回传值（前端原样带回，不让改时间）。

## 7. 数据结构与目录

- **segment**：`{"start": float秒, "end": float秒, "text": str}`，贯穿转录→劣化→TTS→字幕。
- 目录/文件（已加入 `.gitignore`）：
  - `dub_tmp/<task_id>/`：clip/audio/voice/mixed/subs/参考音 等中间产物，任务结束删除。
  - `dubbed_videos/`：最终成品，文件名 `鬼畜配音_<task_id>.mp4`。
  - `voice_config.json`：云端引擎 Key（明天填）。`tts_clone_venv/`：本地 XTTS 环境。
- 依赖：`requirements.txt` 已含 `edge-tts`、`deep-translator`、`faster-whisper`、
  `imageio-ffmpeg`、`av`。

## 8. 音色克隆：已搭好的可插拔框架 + 待联调项（重点）

「音色和原片说话人一致」已做成**可插拔引擎框架**：edge（兜底，不克隆）+ 四个
克隆引擎，前端按 `available()` 点亮，用户选哪个走哪个。**多角色无需显式说话人
分离**——克隆引擎对每句用 `slice_reference()` 切出「本句对应原声」做参考，谁说话
就复刻谁。

### 8.1 各引擎状态与「明天联调」清单
| 引擎 | 状态 | 还差什么 |
|---|---|---|
| `edge` | ✅ 可用 | — |
| `xtts`（本地） | ⚠️ 代码就绪 | 装 `tts_clone_venv`（见 `tts_clone/xtts_worker.py` 头部步骤），首次跑下模型 ~1.8GB；本机有 RTX 4060 Ti 可 GPU 加速 |
| `cosyvoice`（阿里云） | ⚠️ 骨架就绪 | 填 `api_key`；补全 `CosyVoiceEngine.synth`（DashScope 声音复刻 voice_id 流程 + 参考音托管），代码处标 `# 待验证` |
| `iflytek`（讯飞） | ⚠️ 骨架就绪 | 填 `app_id/api_key/api_secret`；补全 `IflytekEngine.synth`（HMAC 鉴权 + 复刻音色 id） |
| `volc`（火山） | ⚠️ 骨架就绪 | 填 `app_id/access_token/cluster`；补全 `VolcEngine.synth`（speaker_id 复刻 + 合成） |

**云端共性坑**：多数云「声音复刻」是「注册音色→拿 voice_id→再合成」，参考音常要求
可访问 URL。逐句本地切片要么先上传/托管，要么整段只注册一个主音色复用（牺牲逐句
多角色）。联调时按各家文档在对应 `synth()` 里定夺，其余代码（对齐/混音/字幕）不动。

### 8.2 联调操作（明天拿到 Key 后）
1. 页面「云端克隆引擎 Key 配置」里填对应字段→保存（写入 `voice_config.json`）。
2. 刷新后引擎下拉里该项点亮，选它跑一段短片。
3. 若报错，按报错改对应引擎的 `synth()`（网络调用都集中在那一个方法里）。
4. 本地 XTTS：先按 worker 头部注释建 venv 装依赖，`available()` 即变 True。

### 8.3 其它
- **转录准确度**：文言腔 + 古人名，Whisper `small` 易错字 → 已用「手工校对」缓解；
  可在 `subtitle/transcriber.py: _get_model()` 把 `"small"` 换 `"medium"` 提升
  （更慢更吃内存）。
- **翻译限流**：Google 网页接口可能限流 → `degrader` 已加重试 + `delay` 限速；
  长片段句子多时整体偏慢。
- **极端语速**：译文远长于时间槽时语音被 `atempo` 大幅加速（受 `TEMPO_MAX` 限幅），
  可能偏快/失真——属预期鬼畜效果，必要时调 `TEMPO_MAX` 或允许片段时长自适应。
- **未做真实端到端验收**：仅用合成短片自动化验证过，未在真实浏览器 + 真实长视频
  手动跑通；首次运行会联网下载 Whisper 模型。

## 9. 改动指引（按需求定位）

| 想改什么 | 改哪里 |
|---|---|
| 让翻译更离谱 / 改遍数上下限 | `degrader.py: PIVOT_LANGS`；前端 `dub.html` 的 rounds 滑块；`webapp` 的 rounds clamp |
| 换/加 edge 预设音色 | `tts.py: DEFAULT_VOICE`；`dub.html` 的 `#voice` 选项 |
| **加新配音/克隆引擎** | 在 `voice_engines.py` 加 `VoiceEngine` 子类并登记到 `_ENGINE_CLASSES`；前端自动列出 |
| **补全某云端引擎** | 改对应 `XxxEngine.synth()`（网络调用都在这一处，标了 `# 待验证`） |
| 调参考音长度/采样率 | `tts.py: slice_reference()` |
| 新字幕位置/颜色/字号 | `burner.py: write_ass()` 默认参数（margin_v/font_size/primary_colour） |
| 背景音量 / 是否保留原音 | `tts.py: mix_into_video(keep_bg, orig_db)`；前端 `keep-bg` 勾选 |
| 转录精度 | `transcriber.py: _get_model()` 模型大小 |
| 阶段文案 | 前端 `dub.js: STAGE_LABELS` |
