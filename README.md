# 视频工具箱

本地运行的视频下载/上传/字幕工具，支持哔哩哔哩、YouTube 两个平台，外加一个通用的中文字幕功能。启动后会自动打开浏览器，所有操作都在网页上完成，没有前后端分离——`main.py` 一个 Flask 进程同时提供页面和接口。

## 功能

- **哔哩哔哩**：扫码登录账号后，粘贴视频链接下载（自动选账号能拿到的最高画质），或把本地视频投稿到自己的账号。
- **YouTube**：下载公开视频（不需要登录），下载时可勾选"自动翻译并烧录中文字幕"和"把文件名翻译成中文"；也可以把本地视频上传到自己的频道（需要先配置 Google OAuth 凭据并登录）。
- **视频字幕**：给任意本地视频文件自动配中文字幕——中文原声直接识别，其他语言先识别再翻译成中文，字幕直接烧录进画面，传到哪个平台都能看到。

## 目录结构

```
  auth.py                 扫码登录、cookie读写
  api.py                  链接解析、视频信息/播放地址
  downloader.py            下载并用ffmpeg合并音视频
  uploader.py              投稿（预上传/分片上传/封面/提交）

  auth.py                 OAuth客户端配置、设备码登录
  downloader.py            yt-dlp封装
  uploader.py              Data API v3上传封装

  transcriber.py           ffmpeg抽音轨 + faster-whisper语音识别
  translator.py            DeepL/免费Google双引擎翻译
  burner.py                生成srt + ffmpeg烧录字幕
templates/              页面模板（index首页选择 / bilibili.html / youtube.html / subtitle.html）
static/                 前端样式和脚本
downloads/              B站下载产物（运行时生成）
youtube_downloads/      YouTube下载产物（运行时生成）
subtitled_videos/       字幕烧录产物（运行时生成）

youtube_token.json       YouTube登录态（运行时生成，不要分享给别人）
deepl_key.json           可选的DeepL翻译API Key（运行时生成）
```

## 运行

```
pip install -r requirements.txt
python main.py
```

会自动打开 `http://127.0.0.1:5000`，看到首页后选择平台。ffmpeg 不需要单独安装，`imageio-ffmpeg` 这个依赖会自带对应系统的二进制。

## 哔哩哔哩

### 登录

打开"哔哩哔哩"页面，点"扫码登录"，会弹出二维码（终端里也会打印一份备用），用 B 站手机 APP 扫码确认。登录态存在本地 `cookies.json` 里，下次打开会自动复用，不用每次都扫。

### 下载视频

粘贴视频的完整链接（如 `https://www.bilibili.com/video/BVxxxxxxxxx/`）或者直接粘贴 BV 号，点下载。会自动用你账号能拿到的最高画质（非大会员账号一般最高 1080P，部分视频可能限制为大会员才能看高画质，会自动降级）。下载完成后历史列表里有"打开文件"链接。

### 投稿到B站

- 选视频文件（必填）。
- 封面可选——不选的话自动用 ffmpeg 截视频第 1 秒当封面。
- 标题、标签必填，简介可选。
- **分区 tid 必填，没有默认值**：B 站分区编号没有可靠的公开对照表，需要你自己先在 [B站创作中心](https://member.bilibili.com) 手动选过一次分区，用浏览器开发者工具（F12 → Network）看请求里的 `tid` 参数，把这个数字填进来。

投稿成功后是"审核中"状态，历史列表会给一个链接，审核通过后才能在 B 站正常看到。

**已知限制**：B 站投稿接口可能触发风控/验证码，目前没法在脚本里自动绕过，出现时页面会直接显示 B 站接口返回的原始错误码和消息。

## YouTube

### 下载视频（不需要登录）

打开"YouTube"页面，下载区块不需要任何登录就能用，直接粘贴视频链接点下载即可。底层用的是 `yt-dlp`，公开视频一般能拿到最高画质。下载完成后历史列表里有两个操作：**"打开文件"**（在浏览器里打开/播放）和**"打开文件所在位置"**（在运行本工具的电脑上弹出文件管理器并选中该文件）。

下载链接下方有两个可选勾选项，可单独勾也可组合：

- **下载后自动翻译并烧录中文字幕**：下载完直接走字幕流程（抽音轨 → faster-whisper 语音识别 → 非中文则翻译成中文 → 把中文字幕烧录进画面），生成的成片存到 `subtitled_videos/`。和"视频字幕"功能同一套引擎，英文/日文/韩文等非中文音频都能翻译成中文。这一步比较耗时（要重新编码整个视频，长视频更慢），首次使用还会自动下载语音识别模型。
- **把文件名翻译成中文**：下载完成后把视频标题翻译成中文，并用中文名重命名文件。翻译失败时会保留原文件名、不影响下载。若两个勾选项同时勾上，烧录字幕后的成片也会用翻译后的中文名。

> 翻译用的引擎和"视频字幕"一致：默认免费 Google 翻译、零配置，配了 DeepL Key 则优先用 DeepL（详见下方"视频字幕"一节）。

### 上传到YouTube频道（需要先配置 + 登录）

YouTube 上传走的是官方 **YouTube Data API v3**，需要你自己有一个 Google Cloud 项目和 OAuth 凭据，这一步必须由你自己在浏览器里操作，无法代做：

1. 打开 [console.cloud.google.com](https://console.cloud.google.com/)，新建或选一个项目。
2. "API和服务 → 库"，启用 **YouTube Data API v3**。
3. "API和服务 → OAuth 同意屏幕"：用户类型选"外部"，发布状态留"测试中"即可（个人用不需要走 Google 验证审核），把自己的 Google 账号加进"测试用户"列表。
4. "API和服务 → 凭据" → 点"创建凭据" → 选"OAuth客户端ID"：
   - **应用类型必须选"电视和有限输入设备"**（登录用的是设备码流程，选"网页应用"或"桌面应用"都会失败）。
   - "名称"随便填一个能认出来的，比如"本地视频工具"，纯粹是给自己看的标签，不影响功能。
   - 点创建。
5. 创建后 Google 会弹出一个对话框，显示两行：
   - **客户端ID**（类似 `123456789-abcdefg.apps.googleusercontent.com`）
   - **客户端密钥**（类似 `GOCSPX-xxxxxxxxxxxxxxxxxxxx`）

   把这两个值原样复制粘贴到 YouTube 页面顶部配置表单的 **Client ID** / **Client Secret** 输入框，点保存即可，以后不会再出现这个表单。如果当时没保存这两个值，之后可以回"凭据"页面点这个客户端ID条目重新看到（客户端密钥可能需要点"显示密钥"或重新生成）。

配置完成后页面会出现"登录Google账号"按钮：

1. 点击后会显示一个二维码和一串验证码（user code）。
2. 用手机扫码打开确认页面，**还需要手动输入这串验证码**才能确认登录——这是 Google 这个登录方式本身的设计，跟 B 站"扫码即过"不完全一样，没法省掉这一步。
3. 确认后页面会自动检测到登录成功，出现上传区块。

登录态（refresh token）存在本地 `youtube_token.json` 里，之后不用重新登录，除非你自己去 Google 账号设置里撤销了授权。

上传时填标题（必填）、简介/标签（可选），隐私状态下拉默认是**私享**（避免手滑公开发布），可以改成"不公开"或"公开"。上传成功后历史列表会给一个 `youtu.be` 链接，可以在自己 YouTube 工作室的"内容"列表里确认。

**已知限制**：YouTube Data API v3 默认每天有调用配额，上传操作消耗比较大，默认配额下每天能传的次数有限（具体数字以你自己 Cloud Console 的"配额"页面为准），超了会报配额错误，页面会原样显示 Google 返回的错误信息，需要的话去 Cloud Console 申请提额。

## 视频字幕

打开首页"视频字幕"卡片，选一个本地视频文件，点"生成字幕"，不需要登录任何账号。处理流程：提取音频 → 语音识别（识别出原声文字+每句话的时间点）→ 如果不是中文就翻译成中文（中文原声跳过这一步）→ 把字幕烧录进画面，生成一个新的 mp4 文件，存在 `subtitled_videos/` 里。

- **字幕是硬字幕**（直接烧录进画面），不是外挂的字幕文件，所以传到B站、YouTube或别的任何平台，字幕都跟着画面一起在，不依赖任何平台自己的字幕功能。代价是要重新编码整个视频，会比较慢，视频越长、电脑性能越差就越慢。
- **语音识别**用的是本地跑的 `faster-whisper`（small模型），不需要联网调用任何付费服务，**第一次使用会自动下载模型文件**（几百MB，需要联网，只有第一次需要等）。
- **翻译默认免费、零配置**：用的是免费的 Google 翻译网页接口，不需要注册任何账号。如果想要更好的翻译质量，可以去 [DeepL官网](https://www.deepl.com/pro-api) 注册一个免费 API Key（每月免费额度 50 万字符），填在页面顶部的配置框里保存一次。配置了 DeepL Key 之后会优先用 DeepL，翻译失败或配额用完会自动切换回免费引擎，不会中断任务。
- **关于"是否需要翻墙"**：实测过免费 Google 翻译接口和 DeepL 接口在国内网络环境下都能直连，不需要额外配置代理，但这不是恒定保证——Google 服务在国内的可访问性跟网络运营商、时间段都有关系，如果某次任务翻译这一步报错连不上，多半是网络问题，重试或者配一个代理再试。

**已知限制**：
- 视频越长处理越久（语音识别 + 重新编码两步都跟时长强相关），暂时没有做"超长视频提前提醒"，建议先用短视频测试效果。
- 语音识别和烧录字幕默认都跑在 CPU 上（代码里没有自动启用 GPU 加速），如果这台机器没有配好 CUDA 环境，速度会明显比有 GPU 加速慢，这是预期之内的，不是出错。

## 数据与隐私

`cookies.json`、`youtube_client.json`、`youtube_token.json`、`deepl_key.json` 都是本地凭据，已经加入 `.gitignore`，不会被提交到代码仓库，但也不要把这几个文件分享给别人（相当于账号密码/API密钥）。
